# Procesos institucionales AS-IS

## 1. Descripción general

Este directorio reúne la transcripción en Markdown de cinco procedimientos institucionales del Colegio Blenkir (matrícula y registro socioeconómico; registro y seguimiento académico; consolidación académica y evaluación de resultados; atención psicológica y tutorial; y gestión directiva de casos académicos críticos). Se transcribió el texto y el cuadro de actividades desde los documentos Word de proceso; no se reprodujeron flujogramas ni figuras Bizagi en esta fase.

## 2. Relación con el proyecto SIDERAE-Blenkir

Estos documentos describen **procesos institucionales actuales** del Colegio Blenkir y sirven como **base documental** para el proyecto **SIDERAE-Blenkir**. No reemplazan el DRS ni certifican el alcance implementado en software.

## 3. Tabla resumen

| Código | Proceso | Área responsable | Relación con SIDERAE-Blenkir |
| --- | --- | --- | --- |
| P-01 | Gestión de matrícula y registro socioeconómico del estudiante | Secretaría / Matrícula | Gestión de estudiantes, variables socioeconómicas y perfil del estudiante |
| P-02 | Gestión de registro y seguimiento académico del estudiante | Área académica / docentes | Registro de notas, asistencia, perfil académico y procesamiento de riesgo |
| P-03 | Gestión de consolidación académica y evaluación de resultados | Coordinación académica | Dashboard, análisis académico, priorización de casos y reportes |
| P-04 | Gestión de atención psicológica y tutorial del estudiante | Psicología y tutoría | Reportes conductuales, intervenciones, seguimiento y cierre de casos |
| P-05 | Gestión directiva de casos académicos críticos | Dirección | Alertas, derivaciones, comunicación familiar, decisiones directivas y trazabilidad |

## 4. Flujogramas y carpeta `diagramas/`

Los flujogramas en **Bizagi** (y análogos BPMN citados en anexos) **no** se documentaron aquí como diagramas reproducidos. La intención es representarlos más adelante en archivos Markdown bajo `docs/procesos/diagramas/`, previa conversión a **Mermaid** a partir de las fuentes gráficas originales.

## 5. Fuentes en `docs/procesos/fuentes/`

Los archivos **.docx** originales permanecen en `docs/procesos/fuentes/` sin modificación y son la referencia prioritaria si hubiera diferencias de transcripción.
